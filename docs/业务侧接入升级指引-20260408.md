# 资产浏览器 SDK 升级指引 — 2026-04-08

## 本次变更：版本展示名称（display_version）

### 背景

此前 `versionId`（如 `v20260408143025`）同时充当内部标识和前端展示名称。本次更新新增 `displayVersion` 字段，允许业务方在创建草稿和发布版本时传入自定义展示名称（如 `v1.2.3`、`2024年春季版`），前端展示优先使用该字段。

`versionId` 保持不变，继续作为系统内部稳定标识。

### 影响范围

| 层级 | 变更内容 | 是否向后兼容 |
|------|----------|-------------|
| Proto 定义 | `AssetVersionSummary` 新增 `display_version = 17` | 是（新增字段） |
| Proto 定义 | `CreateDraftVersionRequest` 新增 `display_version = 6` | 是 |
| Proto 定义 | `PublishDraftVersionRequest` 新增 `display_version = 7` | 是 |
| 数据库 | `asset_versions` 新增 `display_version VARCHAR(255) DEFAULT ''` | 是（需执行迁移） |
| TS 客户端 | `AssetVersionSummary` 接口新增 `displayVersion: string` | 是（空字符串为默认值） |
| TS 客户端 | `createDraft` / `publishDraft` 参数新增 `displayVersion?` | 是（可选参数） |
| React 组件 | 版本选择器、差异标签自动优先展示 `displayVersion` | 是（无需改动） |

### 升级步骤

#### 1. 数据库迁移

执行 SQL 迁移脚本（文件 `migrations/009_add_display_version.sql`）：

```sql
ALTER TABLE asset_versions ADD COLUMN IF NOT EXISTS display_version VARCHAR(255) NOT NULL DEFAULT '';
```

#### 2. 更新依赖版本

确保 `protobuf-typescript-client-gen` 更新到包含 `displayVersion` 的版本，并重新构建：

```bash
cd protobuf-typescript-client-gen && pnpm run build
```

#### 3. 前端代码适配

**无需强制改动**。所有新增字段均为可选或带有空字符串默认值，现有代码无需任何修改即可继续运行。

#### 4.（推荐）使用展示名称

如果希望在前端展示业务版本号：

**使用 TS 客户端直接调用时：**

```typescript
const { versions } = await client.listVersions('my-space', 'my-asset');

// 展示优先级：displayVersion > versionId
const label = version.displayVersion || version.versionId;
```

**使用 React Workspace 组件时：**

组件内部已自动处理展示优先级。版本选择器、侧边栏版本标题、差异标签和底部对比提示均已优先使用 `displayVersion`，无需额外代码。

**创建草稿/发布时传入展示名称：**

```typescript
const draft = await client.createDraft('my-space', 'my-asset', {
  baseVersionId: activeVersionId,
  displayVersion: 'v1.3.0-rc1',   // 可选
});

const published = await client.publishDraft('my-space', 'my-asset', draftId, {
  displayVersion: 'v1.3.0',       // 可选
});
```

### 接口类型变更详情

#### AssetVersionSummary（新增字段）

```typescript
interface AssetVersionSummary {
  // ... 原有字段不变
  displayVersion: string;  // 新增：业务展示用版本号，未设置时为空字符串
}
```

#### createDraft（新增可选参数）

```typescript
client.createDraft(space, assetId, {
  baseVersionId?: string;
  draftVersionId?: string;
  description?: string;
  displayVersion?: string;  // 新增
});
```

#### publishDraft（新增可选参数）

```typescript
client.publishDraft(space, assetId, draftVersionId, {
  versionId?: string;
  description?: string;
  displayVersion?: string;  // 新增
});
```

### 注意事项

- `displayVersion` 为空字符串时，前端应回退到 `versionId` 展示
- `displayVersion` 不参与系统内部版本关联和查找，仅用于 UI 展示
- 已有的 `versionId` 字段含义不变，始终由系统生成
