# 资产浏览器 SDK 升级指引 — 2026-04-16

## 本次变更：修复 Diff 视图文本转义导致的单行比对问题

### 背景

此前版本 Diff 和草稿 Diff 在展示存储中带有 Protobuf 字面量转义的文件时，Monaco DiffEditor 会将整个文件当作单行处理，无法提供行级差异比对。

原因是 `oldPreview` / `newPreview` 中的 `\n` 是两个字符（反斜杠 + n），而非真正的换行符，导致 `DiffEditor` 的 `original` / `modified` 都是单行文本。

### 修复内容

SDK 在 `loadDiffForSelection` 中所有传入 DiffEditor 的文本均经过 `unescapeDiffText` 处理：

```typescript
function unescapeDiffText(text: string): string {
    return needsLiteralUnescape(text) ? unescapeLiteralNewlines(text) : text;
}
```

覆盖场景：

- 版本间 Diff -- `oldPreview` / `newPreview` 直接使用
- 版本间 Diff -- `getDiffEntryDetail` 返回的 `leftText` / `rightText`
- 草稿 Diff -- 以上两种同样覆盖

### 配套后端修改

网关后端同步在 `read_entry_text` 返回前加入了 `maybe_unescape_text` 处理，确保 `similar` 库生成精确的行级 unified diff，并且 API 返回的 `oldPreview` / `newPreview` 是正确的多行文本。

### 业务侧升级方式

无需修改任何业务代码。只需更新依赖并重新构建：

```bash
pnpm install
pnpm build
```
